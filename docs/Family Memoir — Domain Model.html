All FK relationships are nullable unless stated otherwise
Key principle: Location, Event, and Person are all first-class independent entities — create them before any memories exist. All FK relationships on Memory are nullable. Location supports spatial nesting via parent_location_id (a festival stage inside a park) and temporal validity via date_opened / date_closed (a bar that closed and reopened under a new name). Phase 2 canonical layer: two additional tables (canonical_location, canonical_event) plus nullable FK columns on Location and Event will enable multi-user aggregation. Zero changes to the Phase 1 schema — purely additive.
